# DOCX → HTML

Скрипт преобразует текст из DOCX в HTML-фрагмент. Результат не содержит
служебных тегов `<html>`, `<head>` или `<body>` и не использует CSS.

Поддерживаются абзацы, заголовки, маркированные и нумерованные списки,
таблицы, встроенные изображения, выравнивание, жирный текст, курсив,
подчёркивание, зачёркивание, верхний/нижний индекс, шрифт и цвет текста.

## Установка

Нужен Python и Poetry. В папке проекта выполните:

```powershell
poetry install
```

## Конвертация одного файла

Передайте сначала исходный DOCX, затем полный путь к создаваемому HTML:

```powershell
poetry run python main.py .\input\document.docx .\output\document.html
```

Например:

```powershell
poetry run python main.py .\example.docx .\result.html
```

## Конвертация папки

Добавьте флаг `--directory` (или короткий вариант `-d`). В этом режиме первый
путь — папка с DOCX, второй — папка для готовых HTML:

```powershell
poetry run python main.py .\input .\output --directory
```

DOCX из вложенных папок можно обработать флагом `--recursive`:

```powershell
poetry run python main.py .\input .\output --directory --recursive
```

При рекурсивной обработке структура вложенных папок сохраняется в папке
результата. Временные файлы Word с именем, начинающимся на `~$`, пропускаются.

## Уровень заголовка без номера

Стиль Word `Heading 1` / `Заголовок 1` становится `<h1>`, `Heading 2` —
`<h2>` и так далее. Для пользовательского стиля `Heading` или `Заголовок`
без номера по умолчанию используется `<h3>`. Его можно изменить:

```powershell
poetry run python main.py .\input\document.docx .\output\document.html --default-heading-level 2
```

Допустимы значения от 1 до 6.

## Изображения

При конвертации одного DOCX рядом с итоговым HTML автоматически создаётся
папка `Рисунки_<имя DOCX>`. Например, для `document.docx` и результата
`output/document.html` изображения будут лежать здесь:

```text
output/
├── document.html
└── Рисунки_document/
    ├── рис_1_document.png
    └── рис_2_document.jpeg
```

Нумерация начинается с `1` при каждой новой конвертации. Расширение файла
остаётся исходным: PNG сохраняется как `.png`, JPEG — как `.jpeg` или `.jpg`.
Пробелы в названии документа внутри имени изображения заменяются на дефисы:
`рис_6_Технология-бетонных-и-железобетонных-работ.png`.

Для каждого встроенного изображения в HTML добавляется отдельный
центрированный абзац:

```html
<center><img src="https://lms.cbt-center.ru/wp-content/uploads/2026/07/рис_1_Технология-бетонных-и-железобетонных-работ.png" alt="рисунок 1"></center>
```

Имя файла в URL записывается без кодирования: кириллица и пробелы остаются
читаемыми прямо в HTML.

## Автоматическое центрирование

Тегом `<center>` автоматически оборачиваются встроенные изображения,
заголовки Word уровней 2 и 3, а также подписи, начинающиеся с `Рисунок` или
`Рис.` — например, `Рисунок 1 — Схема установки`.

## Использование как модуля

Функции находятся в `tools.py` и не запускают конвертацию при импорте:

```python
from tools import convert_directory, convert_docx_to_html

# Один файл
convert_docx_to_html("input/document.docx", "output/document.html")

# Все DOCX из папки
convert_directory("input", "output", recursive=True)
```

Для своей логики доступны `document_to_html`, `paragraph_to_html` и
`table_to_html`. Это позволяет отдельно добавлять обработку изображений или
других элементов DOCX.
